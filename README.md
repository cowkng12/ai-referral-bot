# AI Referral Bot

Telegram-бот для реферальной системы: пользователь приглашает людей, получает 1 балл за каждого нового реферала и тратит баллы на ИИ-сервисы.

## Возможности

- Реферальная ссылка для каждого пользователя.
- Автоматическое начисление 1 балла за нового приглашенного пользователя.
- Проверка подписки на канал `https://t.me/Omni_Key` при входе.
- Защита от самореферала и повторного начисления одному пользователю.
- Баланс и количество рефералов.
- Магазин ИИ-сервисов с покупкой за баллы.
- Главное меню в стиле `Profile`, `Redeem`, `My Link`, `Progress`, `Store`, `Support`, `Main Channel`.
- Уведомление администраторов о новых покупках.
- Простое JSON-хранилище в `data/db.json`.

## Запуск

Быстрый способ: открой `run_bot.bat`. Бот запустится прямо в этом окне.

Если нужна локальная панель управления, открой `run_panel.bat`, затем перейди на `http://localhost:3000` и нажми `Запустить бота`.

1. Создай бота через [@BotFather](https://t.me/BotFather) и получи токен.
2. Установи зависимости:

```bash
npm install
```

3. Скопируй `.env.example` в `.env` и заполни значения:

```env
BOT_TOKEN=your_telegram_bot_token
ADMIN_IDS=123456789
```

`ADMIN_IDS` можно указать через запятую: `123,456`.
`SHOP_URL` - ссылка на магазин `@OminiKey_bot`, она показывается в разделе `Store`.
`SUPPORT_URL` - ссылка на поддержку `@OmniKeySUPPORT`.
`MAIN_CHANNEL_URL` - ссылка на основной канал.
`MAIN_CHANNEL_USERNAME` - username канала для проверки подписки. Чтобы проверка работала стабильно, добавь бота администратором в канал.

4. Запусти бота:

```bash
npm start
```

Или запусти локальную панель управления:

```bash
npm run panel
```

После этого открой `http://localhost:3000` и запускай/останавливай бота кнопками.

## Деплой на Render

Проект подготовлен для запуска на Render как `Web Service`. Сервис слушает HTTP-порт Render для health-check, а Telegram-бот работает в том же процессе через long polling.

1. Загрузи проект в GitHub/GitLab.
2. В Render создай новый `Blueprint` из репозитория или `Web Service` вручную.
3. Если создаешь вручную, укажи команды:

```bash
npm install
npm run render:start
```

4. Добавь переменные окружения:

```env
BOT_TOKEN=your_telegram_bot_token
ADMIN_IDS=123456789
DATA_DIR=/var/data
SHOP_URL=https://t.me/OminiKey_bot
SUPPORT_URL=https://t.me/OmniKeySUPPORT
MAIN_CHANNEL_URL=https://t.me/Omni_Key
MAIN_CHANNEL_USERNAME=@Omni_Key
```

5. Подключи persistent disk с mount path `/var/data`, чтобы файл `db.json` не удалялся при перезапуске сервиса.

Локальную панель `npm run panel` на Render запускать не нужно: Web Service использует `src/server.js`, который запускает бота и HTTP health-check.

## Команды

- `/start` - запуск бота и обработка реферальной ссылки.
- `/balance` - баланс и количество рефералов.
- `/link` - личная реферальная ссылка.
- `/shop` - магазин ИИ-сервисов.
- `/profile` - профиль пользователя.
- `/progress` - прогресс по баллам и покупкам.
- `/support` - ссылка на поддержку.
- `/channel` - ссылка на основной канал.

## Настройка сервисов

Список товаров находится в `src/bot.js` в массиве `defaultDb.services`. Бот показывает `🔒`, если пользователь еще не получал товар, и `🔑`, если товар уже покупался.

## Как работает начисление

Если новый пользователь запускает бота по ссылке вида `https://t.me/BOT_USERNAME?start=ref_USER_ID`, пригласивший пользователь получает 1 балл. Начисление происходит только один раз для каждого нового пользователя.
